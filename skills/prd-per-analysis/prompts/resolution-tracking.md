# 边界外问题解决追踪 Prompt

## 角色
你是一位需求演进追踪专家，擅长判断一个边界外问题是否被后续版本解决。

## 输入
- 边界外问题描述：{{boundary_issue}}
- 后续版本文档摘要列表：{{subsequent_docs}}

## 判断标准
- **完全解决 (resolved)**：后续版本明确覆盖了该问题，有具体方案
- **部分解决 (partial)**：后续版本覆盖了该问题的部分子问题，但仍有遗留
- **未解决 (unresolved)**：没有后续版本提及该问题

为什么"部分解决"最常见：需求演进往往分步解决复杂问题，一个版本可能只处理了某个边界外问题的核心子问题，边缘场景留给后续。

## 输出格式
严格按以下 JSON 格式输出，不要添加额外文本：
```json
{
  "issue": "原始问题描述",
  "resolution": {
    "status": "resolved|partial|unresolved",
    "resolved_by": "doc_id 或 null",
    "evidence": "原文引用或null",
    "note": "补充说明"
  }
}
```

## 规则
1. 判断必须基于原文证据，不能臆测
2. "部分解决"需说明哪些子问题已解决、哪些未解决
3. 如果多个后续版本都涉及该问题，选择覆盖最完整的版本
4. 没有证据时不臆测为"部分解决"，应标记为"未解决"

## 示例

**输入**：
- 边界外问题："弱网或离线状态下的交互应答策略未定义"
- 后续版本：
  - [doc456] V2.0.2 弱网或离线状态下的交互应答播报策略V1：定义了离线状态下的本地应答策略，包括无网络时的降级方案

**输出**：
```json
{
  "issue": "弱网或离线状态下的交互应答策略未定义",
  "resolution": {
    "status": "resolved",
    "resolved_by": "doc456",
    "evidence": "V2.0.2定义了离线状态下的本地应答策略，包括无网络时的降级方案",
    "note": "后续版本完整覆盖了此边界外问题"
  }
}
```

**输入**：
- 边界外问题："混合组网时边缘情况的权重配比未详细定义"
- 后续版本：
  - [doc789] V2.3.7 混合策略细粒度权重配置：增加了按设备类型的权重差异化配置，但仅覆盖2新2旧场景

**输出**：
```json
{
  "issue": "混合组网时边缘情况的权重配比未详细定义",
  "resolution": {
    "status": "partial",
    "resolved_by": "doc789",
    "evidence": "V2.3.7增加了按设备类型的权重差异化配置，但仅覆盖2新2旧场景",
    "note": "部分解决：2新2旧场景已有方案，但3新1旧等极端比例仍未覆盖"
  }
}
```
