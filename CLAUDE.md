# 项目指引

1. 每次对话的最后一个步骤，必须检查本次对话是否涉及任务完成（包括但不限于：bug 修复、功能开发、代码审查、测试数据准备、文档更新等）。检查完成后：
   - 将新增条目、状态变更记录到 `task-list.md` 中，确保任务列表与实际进度同步；
   - 在对话最后一条回复中通知用户：已记录哪些条目。

2. 每次完成 bug 修复或功能开发时，都要为对应的 bug 或功能点设计至少一个自动化测试脚本，并至少跑一遍相关的自动化测试；如果存在全量自动化测试入口，收尾前优先补一次全量回归验证。

3. 遇到不理解的问题，主动查找官方文档和权威来源，不要凭猜测给出答案。

4. **数据与代码分离原则**：runtime/ 目录存放所有运行时数据（数据库、上传文件、日志、结果），该目录已 git-ignored。不要将任何运行时数据或用户数据提交到 git。

5. **开源合规原则**：不要在代码中硬编码品牌名称、内部域名、特定企业标识。所有品牌元素应通过配置文件或环境变量注入。
