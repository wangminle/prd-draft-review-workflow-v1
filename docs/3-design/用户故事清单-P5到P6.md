## 👤 P5.A：个人空间与 Agent

### US-31 个人资料与团队资料切换

**角色**：任意已登录用户  
**操作**：进入团队空间 → 资料库 Tab → 点击"团队资料/我的资料"切换按钮  
**验收**：
- 默认显示"团队资料"列表（owner_type=workspace, visibility=team）
- 切换到"我的资料"后显示个人私有资料列表（owner_type=user, visibility=private）
- 个人资料上传后默认不进入团队知识库检索
- 上传按钮文案随 scope 变化："上传资料" / "上传个人资料"
- 个人资料可删除（无需管理员权限），团队资料删除需 owner/admin 权限
- 个人资料详情页和下载功能正常工作  
**对应阶段**：P5.A.1

---

### US-32 个人 Agent 设置

**角色**：任意已登录用户  
**操作**：点击顶栏用户名 → 下拉菜单选择"Agent 设置"  
**验收**：
- 弹出模态框显示个人 Agent 配置
- 可修改 Agent 名称、默认访问范围（个人/团队）、启用/禁用状态
- 保存后设置生效
- 非管理员也可操作自己的 Agent 设置  
**对应阶段**：P5.A.2 + P5.A.4

---

### US-33 Agent 对话请求通知

**角色**：Agent 所有者  
**操作**：他人向你的 Agent 提问后  
**验收**：
- 收到通知（铃铛显示红点/数字）
- 通知类型为"Agent 对话请求"，显示提问摘要
- 点击"查看"跳转到智能对话页面  
**对应阶段**：P5.A.3

---

## 💬 P5.B：消息中心

### US-34 消息中心浏览

**角色**：任意已登录用户  
**操作**：点击顶栏通知铃铛图标  
**验收**：
- 下拉面板显示通知列表，含未读/已读/归档三个 Tab
- 未读通知有蓝点标记
- 通知按时间倒序排列
- 点击"查看"跳转到对应页面（审查/管理后台/智能对话）
- "全部已读"按钮可一键标记所有通知为已读  
**对应阶段**：P5.B.1 + P5.B.3

---

### US-35 消息类型区分

**角色**：收到多种通知的用户  
**操作**：在消息中心查看不同类型通知  
**验收**：
- 可区分以下通知类型：审查请求、审批请求、评论回复、@提及、Agent 对话请求
- 每种类型有不同的标题和图标提示
- 审查相关通知点击后跳转到审查页面
- Agent 审批通知点击后跳转到管理后台
- Agent 对话通知点击后跳转到智能对话  
**对应阶段**：P5.B.2

---

## 📝 P5.C：评论与提及

### US-36 @提及通知

**角色**：被提及的用户  
**操作**：他人在评论中输入 @你的用户名 → 你收到通知  
**验收**：
- 评论创建时自动检测 @username 格式
- 被提及者收到 mention 类型通知
- 通知标题显示谁提到了你
- 点击通知可跳转到对应评论区域  
**对应阶段**：P5.C.1

---

### US-37 评论 resolve

**角色**：审查参与者  
**操作**：在审查问题评论上点击"解决"按钮 → 选择 resolve 方式  
**验收**：
- 可标记评论为"已解决"(resolved)或"强制通过"(forced_pass)
- 已解决的评论显示 resolved 标记和解决人
- resolve 操作触发通知给评论作者
- 审查问题行级评论支持 resolve 标记  
**对应阶段**：P5.C.2 + P5.C.3

---

## 📊 P6.A：成本与质量仪表盘

### US-38 成本统计查看

**角色**：管理员  
**操作**：访问 `/api/governance/cost/daily` 查看每日成本统计  
**验收**：
- 返回按日期、模型分组的 LLM 调用统计
- 包含调用次数、输入/输出 token 数、总耗时
- 支持日期范围筛选
- 可手动触发成本聚合（从 JSONL 日志）
- 查看汇总统计（总调用数、总 token 数）  
**对应阶段**：P6.A.1

---

### US-39 质量趋势查看

**角色**：管理员  
**操作**：访问 `/api/governance/quality/weekly` 查看每周质量统计  
**验收**：
- 返回按周的质量统计
- 包含平均评分、审查数量
- 支持周范围筛选  
**对应阶段**：P6.A.2

## 🔧 P6.B：Skill/Agent 治理

### US-40 Skill 包管理

**角色**：管理员  
**操作**：访问治理页面查看 Skill 列表 → 更新 Skill 状态  
**验收**：
- 列出所有注册 Skill（id、skill_id、名称、描述、状态、版本）
- 可更新 Skill 状态为 published/draft/deprecated
- Skill 回归测试框架自动验证 SKILL.md 存在且包含必要章节  
**对应阶段**：P6.B.1 + P6.B.2

---

### US-41 Agent 退役

**角色**：管理员  
**操作**：将 disabled 的 Agent 退役（归档）  
**验收**：
- 只有 disabled 状态的 Agent 才能退役
- 有活跃运行的 Agent 无法退役（返回 422 错误）
- 退役后状态变为 archived  
**对应阶段**：P6.B.3

---

### US-42 权限审计报告

**角色**：管理员  
**操作**：访问 `/api/governance/permissions/audit` 查看权限审计报告  
**验收**：
- 返回 workspace 成员角色列表
- 返回 Agent 授权变更记录
- 报告可导出为 JSON  
**对应阶段**：P6.B.4

---

## ⚙️ P6.C：配额管理

### US-43 Workspace 配额设置

**角色**：管理员  
**操作**：设置 Workspace 月度配额 → 查看当前配额  
**验收**：
- 可配置月度 token 限制、月度费用限制
- 可设置预警阈值百分比（默认 80%）
- 可设置硬限制动作：通知(notify)或阻断(block)
- 查看配额时返回完整配置信息  
**对应阶段**：P6.C.1 + P6.C.3

---

### US-44 配额拦截

**角色**：使用 AI 功能的用户  
**操作**：当月 token 使用量超过配额硬限制  
**验收**：
- 超过预警阈值时收到通知提醒
- 超过硬限制且 hard_limit_action=block 时，AI 调用被拒绝
- 管理员可调整配额恢复服务  
**对应阶段**：P6.C.2
