# 用户故事清单（P0 ~ P4）

> 版本：V0.3.0 | 日期：2026-06-12
> 覆盖范围：P0 团队资料库 → P1 权限管理 → P2 知识检索 → P3 Agent 模式 → P4 协作审查
> 共 30 条用户故事，按页面和流程组织，可逐条在浏览器中操作验证。

---

## 🏠 基础导航与认证

### US-01 注册与登录

**角色**：新用户  
**操作**：注册新账号 → 登录系统 → 进入审查工作台，顶栏右侧显示页面切换链接  
**验收**：登录成功后顶栏显示用户名，顶栏右侧导航可切换到其他页面（智能对话、团队空间）；管理员还可看到"管理后台"链接。  
**对应阶段**：基座

---

### US-02 顶栏品牌与版本

**角色**：任意已登录用户  
**操作**：观察顶栏左侧  
**验收**：显示产品名"AI产品需求初审"和版本号"Ver. 0.3.0"；右侧显示用户名和退出按钮。  
**对应阶段**：基座

---

## 📚 P0：团队资料库与项目引用

### US-03 上传资料到团队资料库

**角色**：团队成员  
**操作**：进入"团队空间"页面 → 点击上传按钮 → 上传一个 DOCX 文件  
**验收**：上传完成后资料列表出现新条目，显示文件名、类型（emoji）、版本、状态（active）。  
**对应阶段**：P0.A / P0.B

---

### US-04 查看资料详情与下载

**角色**：团队成员  
**操作**：在资料列表点击某条资料  
**验收**：展开详情面板，显示文件名、content hash、标签、正文预览（前 500 字符）；点击"下载"按钮可下载原文件。  
**对应阶段**：P0.B

---

### US-05 删除资料

**角色**：owner / admin  
**操作**：在资料列表点击某条资料的删除按钮  
**验收**：资料被软删除（状态变为 deleted），列表不再显示该条目；普通成员无删除按钮。  
**对应阶段**：P0.B

---

### US-06 项目引用资料

**角色**：项目创建者  
**操作**：在审查项目页点击"引用资料"按钮 → 弹出资料选择器 → 勾选若干条资料 → 设置引用类型（上下文/参考/背景） → 确认  
**验收**：项目页显示引用的资料列表，每条显示引用类型；审查启动时自动冻结版本快照。  
**对应阶段**：P0.C

---

## 👥 P1：权限与成员管理

### US-07 查看团队成员列表

**角色**：团队成员  
**操作**：进入"团队空间" → 切换到"团队成员"Tab  
**验收**：显示所有成员的用户名、角色（owner/admin/member/viewer）、状态（活跃/已停用），包括已停用成员。  
**对应阶段**：P1.A

---

### US-08 变更成员角色

**角色**：owner / admin  
**操作**：在成员列表通过下拉菜单将某成员从 member 改为 admin → 弹出确认框 → 确认  
**验收**：角色即时更新；owner 降级时有二次确认弹窗。  
**对应阶段**：P1.A

---

### US-09 停用/恢复成员

**角色**：owner / admin  
**操作**：点击某成员的"停用"按钮 → 确认  
**验收**：成员状态变为"已停用"；停用成员无法访问项目和资料；点击"恢复"可重新激活。  
**对应阶段**：P1.A

---

### US-10 修改团队空间信息

**角色**：owner / admin  
**操作**：在团队空间设置区修改团队名称和描述 → 保存  
**验收**：名称和描述即时生效；普通成员只有只读视图。  
**对应阶段**：P1.A

---

## 💬 P2：知识检索与对话

### US-11 普通对话

**角色**：任意已登录用户  
**操作**：进入"智能对话"页面 → 选择模型 → 输入问题 → 发送  
**验收**：AI 以流式方式回复；对话记录保存到左侧列表；可切换历史对话继续聊天。  
**对应阶段**：P2 基座

---

### US-12 开启知识库引用

**角色**：任意已登录用户  
**操作**：在对话页面点击"引用资料"开关 → 输入与团队知识相关的问题 → 发送  
**验收**：AI 回复中包含引用卡片（来源标题、章节、置信度高/中/低）；正文区分"知识库回答"和"模型推理"段落；底部显示引用资料标签。  
**对应阶段**：P2.C

---

### US-13 切换历史对话

**角色**：任意已登录用户  
**操作**：在左侧对话列表点击某条历史对话  
**验收**：右侧加载完整聊天记录；上下文面板同步加载该对话的上下文项；可继续发送消息。  
**对应阶段**：P2 基座

---

### US-14 上下文管理

**角色**：任意已登录用户  
**操作**：在对话中上传文件或粘贴 URL → 打开上下文面板 → 查看各上下文项 → 切换启停状态  
**验收**：上下文面板列出所有附件和 URL；禁用的项不注入对话；发送消息时 AI 能感知启用的上下文。  
**对应阶段**：P2 基座

---

## 🤖 P3：Agent 模式与工具审批

### US-15 开启 Agent 模式

**角色**：任意已登录用户  
**操作**：在对话页面点击"Agent"开关 → 输入任务目标（如"帮我搜索XXX"） → 发送  
**验收**：Agent 自主执行工具调用；回复中显示执行步骤（plan/tool/observe/respond）和工具调用轨迹（工具名、状态、耗时、风险级别图标）。  
**对应阶段**：P3.E

---

### US-16 工具调用审批

**角色**：管理员 / 审批人  
**操作**：Agent 执行高风险工具时触发审批 → 进入管理后台 → Agent 设置 → 在"待审批请求"区域查看待审批请求 → 点击"批准"或"拒绝"  
**验收**：批准后 Agent 继续执行；拒绝后 Agent 中止该工具调用；审批记录可追溯。  
**对应阶段**：P3.D

---

### US-17 Agent 配置

**角色**：管理员  
**操作**：进入管理后台 → Agent 设置页 → 修改 Agent 名称、系统策略、允许工具列表 → 保存  
**验收**：配置保存后即时生效；下次 Agent 运行时使用新配置。  
**对应阶段**：P3.A / P3.B

---

### US-18 MCP 服务器配置

**角色**：管理员  
**操作**：通过 API 配置 MCP 服务器（名称、URL、连接方式）和工具策略  
**验收**：MCP 服务器配置保存成功；Agent 可调用该服务器上的工具；需要审批的工具触发审批流程。  
**对应阶段**：P3.C  
> ⚠️ 当前仅后端 API 支持，管理后台暂无对应 UI 页面。

---

## 🔍 审查核心流程（P0/P2 基座）

### US-19 创建审查项目并上传文档

**角色**：任意已登录用户  
**操作**：进入"审查模式"页面 → 点击"新建项目" → 输入名称 → 上传 DOCX 文件  
**验收**：项目创建成功，出现在项目列表；文档出现在项目文档列表，显示文件名和类型。  
**对应阶段**：P0 基座

---

### US-20 执行快速审查

**角色**：项目创建者  
**操作**：选择审查模式（quick / review / full 等） → 选择模型 → 点击"开始审查"  
**验收**：页面显示流式进度（分类 → 逐篇分析 → 体系评审 → 报告）；审查完成后展示结果。  
**对应阶段**：P0 基座

---

### US-21 查看审查结果

**角色**：项目创建者  
**操作**：审查完成后在结果区域浏览  
**验收**：能看到概览分类、逐篇分析（核心问题 / 评分 / 专家意见 / 边界外问题）、体系评审（业务价值 / 行动计划 / 竞品定位 / 产品策略 / 技术演进），各区块可折叠展开。  
**对应阶段**：P0 基座

---

## 🤝 P4：协作审查

### US-22 发起协作审查

**角色**：项目创建者  
**操作**：审查完成后在结果区域点击"发起协作审查" → 填写审查目标 → 指定审批人（Approver） → 确认  
**验收**：协作审查请求创建成功，状态为"待审批"；审批人收到通知。  
**对应阶段**：P4.A

---

### US-23 审批人审批/驳回

**角色**：审批人（Approver）  
**操作**：收到通知后在审查页面查看协作审查请求 → 点击"通过"或"驳回"并填写意见  
**验收**：通过后请求状态变为"已通过"；驳回后状态变为"已驳回"；发起人收到审批结果通知。  
**对应阶段**：P4.A

---

### US-24 驳回后重新提交

**角色**：发起人  
**操作**：审查被驳回后点击"重新提交"  
**验收**：系统自动创建新一轮并继承上轮审批人；请求状态回到"待审批"；审批人收到新通知。  
**对应阶段**：P4.A

---

## 📦 P4：产物与讲解

### US-25 创建与确认产物

**角色**：项目创建者  
**操作**：在审查结果区域点击"创建产物" → 选择类型（讲解稿 / 流程图 / HTML展示 / SVG概要） → 填写内容 → 保存  
**验收**：产物创建为 draft 状态；点击"确认"冻结为 confirmed 状态；confirmed 状态不可修改内容；点击"取消确认"回退为 draft。  
**对应阶段**：P4.B

---

### US-26 进入讲解准备模式

**角色**：项目创建者  
**操作**：审查完成后点击"准备讲解"  
**验收**：页面切换到智能对话，欢迎区域显示"讲解准备模式"提示；AI 自动注入审查结果和项目知识上下文；可与 AI 迭代优化讲解稿。  
**对应阶段**：P4.B

---

## 🔔 P4：通知与评论

### US-27 通知铃铛与未读数

**角色**：任意已登录用户  
**操作**：观察任意页面顶栏右侧  
**验收**：看到通知铃铛图标和未读数角标（红色数字）；点击铃铛展开下拉面板，显示通知列表；面板有"未读/已读/归档"三个 Tab 可切换。  
**对应阶段**：P4.D

---

### US-28 通知跳转与标记已读

**角色**：任意已登录用户  
**操作**：在通知列表点击某条通知的"查看"按钮  
**验收**：页面跳转到对应对象（审查请求 / Agent 审批 / 产物详情等）；同时该通知标记为已读，角标数字减 1；点击"全部已读"批量标记。  
**对应阶段**：P4.D

---

### US-29 创建评论与回复

**角色**：任意已登录用户  
**操作**：在审查结果区域的评论区输入评论内容 → 点击发送；点击某条评论的"回复" → 输入回复内容 → 发送  
**验收**：评论即时出现在列表中；回复显示在原评论下方缩进；显示作者和时间。  
**对应阶段**：P4.D

---

### US-30 @提及通知

**角色**：任意已登录用户  
**操作**：在评论中输入 @用户名 → 提交评论  
**验收**：被提及的用户收到一条通知；通知内容显示提及者与评论摘要；点击通知可跳转到评论所在位置。  
**对应阶段**：P4.D
